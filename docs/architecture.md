# Архитектура проекта: Система много-зонного полива

## 1. Аппаратная платформа
* Контроллер: Raspberry Pi Pico (микроконтроллер на базе RP2040)[span_2](start_span)[span_2](end_span).
* Язык программирования: MicroPython[span_3](start_span)[span_3](end_span).

## 2. Используемые компоненты (Wokwi)
Для реализации полноценного прототипа в симуляции планируются следующие компоненты[span_4](start_span)[span_4](end_span):
* Датчики влажности: 3 аналоговых потенциометра (SIG -> ADC0, ADC1, ADC2)[span_5](start_span)[span_5](end_span).
* Датчик уровня бака: Двухпозиционный переключатель (Slide Switch)[span_6](start_span)[span_6](end_span).
* Исполнительные механизмы: Блок из 4-х реле (3 для клапанов зон и 1 для главного насоса)[span_7](start_span)[span_7](end_span).
* Интерфейс пользователя: OLED-дисплей SSD1306 (I2C)[span_8](start_span)[span_8](end_span).
* Модуль времени: DS1307 RTC (I2C) для работы по расписанию[span_9](start_span)[span_9](end_span).

## 3. Схема подключений (Интерфейсы)
* I2C Bus: Пины GP0 (SDA) и GP1 (SCL) используются совместно для дисплея и часов реального времени[span_10](start_span)[span_10](end_span).
* Analog Inputs (ADC):
    * GP26 — Зона 1
    * GP27 — Зона 2
    * GP28 — Зона 3[span_11](start_span)[span_11](end_span).
* Digital I/O:
    * GP14 — Вход датчика бака[span_12](start_span)[span_12](end_span).
    * GP16, GP17, GP18, GP19 — Выходы управления реле[span_13](start_span)[span_13](end_span).

## 4. Логика защиты
Система предусматривает программную блокировку включения насоса при низком уровне воды в баке (сигнал с переключателя) и защиту от слишком частого срабатывания реле (гистерезис)[span_14](start_span)[span_14](end_span).
